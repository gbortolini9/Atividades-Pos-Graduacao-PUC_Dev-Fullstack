# Arquitetura Client Side

Este diretório contém as atividades desenvolvidas para a disciplina **Arquitetura Client-Side** do curso de Pós-Graduação em Desenvolvimento Full Stack da PUC.

O objetivo das atividades é explorar conceitos fundamentais da arquitetura de aplicações web modernas, incluindo **Single Page Applications (SPA)**, **componentização**, **micro frontends** e o uso de **frameworks baseados em React**.

---

# Tecnologias utilizadas

- HTML5
- CSS3
- JavaScript
- React
- Next.js

---

# Estrutura das atividades

## Atividade 1 — Micro Frontends e App Shell

Demonstração de uma arquitetura baseada em **micro frontends**, utilizando um **container principal (App Shell)** responsável por carregar diferentes módulos da aplicação.

Conceitos abordados:

- Arquitetura App Shell
- Micro Frontends
- SPA (Single Page Application)
- Roteamento client-side
- Separação de responsabilidades

Exemplo prático de criação de **componentes visuais utilizando React e JSX**, demonstrando como componentes podem receber dados externos através de **props**.

Conceitos abordados:

- Componentização
- JSX
- Props

## Atividade 3 — Estrutura de aplicação com Next.js

Exemplo de aplicação estruturada utilizando o framework **Next.js**, demonstrando como frameworks ajudam a organizar aplicações React mais complexas.

Conceitos abordados:

- Frameworks para React
- Estrutura de projeto
- Roteamento por páginas
- Componentização
- Separação de responsabilidades

---

# Objetivo das atividades

As atividades têm como objetivo demonstrar diferentes abordagens de arquitetura client-side, desde soluções simples em JavaScript até o uso de frameworks modernos como **React** e **Next.js**, permitindo compreender como aplicações web podem ser estruturadas de forma modular, escalável e reutilizável.
